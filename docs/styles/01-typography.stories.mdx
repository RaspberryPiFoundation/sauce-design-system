# Typography

## Fonts

Include this in your application `<head>`. The following example includes expanded whitespace so you can see what it is, but feel free to minify it in your application.

```html
<link crossorigin href="https://fonts.gstatic.com" rel="preconnect" />
<link
  as="style"
  href="https://fonts.googleapis.com/css?family=Rubik:300,400,500,700&display=swap"
  rel="preload"
/>
<link
  href="https://fonts.googleapis.com/css?family=Rubik:300,400,500,700&display=swap"
  media="print"
  onload="this.media='all'"
  rel="stylesheet"
/>
<noscript>
  <link
    href="https://fonts.googleapis.com/css?family=Rubik:300,400,500,700&display=swap"
    rel="stylesheet"
  />
</noscript>
```

This HTML fragment makes use of Resource hinting to provide the fastest download of the typefaces we use. It looks counter-intuitive to include more HTML to achieve this aim, but it helps the browser correctly prioritise and handle the request.

For a deep-dive explanation of what's happening, read Harry Roberts' article "[The Fastest Google Fonts](https://csswizardry.com/2020/05/the-fastest-google-fonts/)".

## Font Family

| Name          | Value                                  |
| ------------- | -------------------------------------- |
| `$serif`      | 'Rubik', serif;                        |
| `$sans-serif` | 'Rubik', sans-serif;                   |
| `$monospace`  | 'Lucida Console', 'Monaco', monospace; |

```scss
@use '/src/tokens/font-family';

.some-selector {
  font-family: font-family.$monospace;
}
```

## Font Size

Try to use the `font-size` mixin module where possible. These named sizes have different values depending on the screen size, so we can employ the correct font-size for a particular screen size. They use [CSS's `clamp`]() to provide scaling styles without the need for `@media` queries.

The clamped values attempt to set a "target" value based on the size we would expect to see in a browser exactly 1000px wide:

| Token          | "target" |
| -------------- | -------- |
| `$rollerskate` | 1.6rem   |
| `$skateboard`  | 2rem     |
| `$scooter`     | 2.4rem   |
| `$bike`        | 3rem     |
| `$moped`       | 4rem     |
| `$car`         | 5rem     |
| `$van`         | 7rem     |

To achieve this with `clamp`, we can state that to achieve `$scooter: 2.4rem` at 1000px browser width, we need the "target" size to equal `2.4vw` (1000px &times; 2.4vw = 24px = 2.4rem).

We also instruct the browser to cap the values at a minimum and maximum size. Our values currently work out we will use the "min" value between 0 and 800px width, the "target" value between 800px and 1200px, and the "max" value from 1200px upwards:

| Token          | Min     | Target     | Max     | Scale      |
| -------------- | ------- | ---------- | ------- | ---------- |
| `$rollerskate` | 1.28rem | 1.6vw      | 1.92rem | &times;0.8 |
| `$skateboard`  | 1.6rem  | 2vw        | 2.4rem  | &times;1   |
| `$scooter`     | 1.92rem | 2.4vw      | 2.88rem | &times;1.2 |
| `$bike`        | 2.4rem  | 3vw        | 3.6rem  | &times;1.5 |
| `$moped`       | 3.2rem  | 4vw        | 4.8rem  | &times;2   |
| `$car`         | 4rem    | 5vw        | 6rem    | &times;2.5 |
| `$van`         | 5.6rem  | 7vw        | 8.4rem  | &times;3.5 |

```scss
@use '/src/mixins/font-size';

.some-selector {
  @include font-size.rollerskate;
  @include font-size.skateboard;
  @include font-size.scooter;
  @include font-size.bike;
  @include font-size.moped;
  @include font-size.car;
  @include font-size.van;
}
```

## Font Weight

| Name       | Value |
| ---------- | ----- |
| `$light`   | 300   |
| `$regular` | 400   |
| `$medium`  | 500   |
| `$bold`    | 700   |

```scss
@use '/src/tokens/font-weight';

.some-selector {
  font-weight: font-weight.$light;
}
```

## Line Heights

| Name       | Value |
| ---------- | ----- |
| `$loose`   | 1.6   |
| `$regular` | 1.4   |
| `$cosy`    | 1.2   |
| `$tight`   | 1.1   |

Use like this:

```scss
@use '/src/tokens/line-height';

.some-selector {
  line-height: line-height.$cosy;
}
```
